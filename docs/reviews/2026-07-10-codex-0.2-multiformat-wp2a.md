# Review: Auto SVGA 0.2-WP2A Lottie JSON Inspection

## 1. Summary

Status: `WP2A Implementation Ready`, pending independent Code Review.

Implemented a no-new-dependency Lottie JSON inspection service for
`Auto SVGA 0.2.0-alpha.1`. The service uses built-in JSON parsing only, is
callable only with the explicit `0.2-multiformat-preview-wp2a` gate, and
normalizes Lottie metadata into existing `MotionAssetInfo` contracts.

This does not implement or expose Lottie playback, renderer integration,
thumbnails, UI/open flow, editing, save, export, `.lottie` package support,
Windows, AEB, package promotion, or production support.

## 5. Implementation

### Inspection Service

- Added `LottieJsonInspectionService.inspect()`.
- Requires the explicit `0.2-multiformat-preview-wp2a` gate before source
  reads.
- Uses bounded reads with the same 256 KiB sample cap as WP1; range-readable
  sources use `readRange(0, 262144)`.
- Parses JSON with built-in `JSON.parse` only.
- Returns `WorkbenchResult<MotionAssetInfo>` and typed
  `LottieJsonInspectionIssue` feedback.

### Normalized Metadata

The service normalizes:

- Lottie version.
- Canvas dimensions.
- `fr`, `ip`, `op`, duration frames, and duration milliseconds.
- Layer count and common layer metadata.
- Image assets as metadata-only `MotionResourceInfo` entries.
- Precomp assets as vector metadata resources.
- Text layers as replaceable text candidates.
- Font list entries as font resources.
- Unsupported feature markers as warning issues and metadata.

### External Image References

External image references are not read. WP2A accepts only deterministic
relative paths and records them as metadata.

Fail-closed cases:

- absolute macOS or Windows paths;
- parent traversal;
- file URLs and other URI schemes;
- missing image references;
- duplicate asset ids;
- missing layer `refId`;
- layer `refId` pointing at a missing asset.

### Capability Baseline

The Lottie `parse` capability is now marked `partial` with a boundary that it
is metadata-only JSON inspection. Lottie playback, replaceable content,
conversion, export, spec check, and performance check remain non-current.

## 10. Risks And Limitations

- The 256 KiB prefix is a bounded inspection limit. Larger Lottie JSON remains
  a precondition failure until a later approved read strategy exists.
- External images are only metadata. WP2A does not prove image existence,
  dimensions from decoded bytes, alpha, thumbnails, or replacement readiness.
- Minimal text and font extraction is structural. It does not prove renderer
  text behavior.
- Unsupported markers are intentionally conservative and incomplete. Code
  Review should treat this as inspection scaffolding, not feature coverage.
- The service has no production call site and no visible client entry.

## 11. Rollback

Rollback is a single source commit revert for WP2A. The new service has no
production call site, state migration, dependency, generated asset, package, or
owner-visible behavior.

## 13. Project Retrospective

- Value assessment: High.
- Cost drivers: Lottie JSON has many optional structures; external image paths
  needed strict fail-closed validation without reading assets.
- Avoidable costs: none significant; WP1 detector and path-redaction patterns
  made WP2A narrower.
- Product lesson: metadata inspection can advance 0.2 readiness without
  changing 0.1 visible support.
- Technical lesson: external asset paths should be treated as untrusted data
  until a host resolver owns path policy.
- Design / interaction lesson: text candidates can be normalized for future UI
  without introducing visible controls now.
- Process lesson: stop at the renderer dependency gate after parser/inspection
  acceptance.
- Token usage source: unavailable.
