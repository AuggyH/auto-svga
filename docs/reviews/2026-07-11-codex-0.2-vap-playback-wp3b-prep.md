## Implementation Preparation

Added `VapPlaybackPreparationService`, a dependency-free preparation layer over
the accepted VAP inspection `MotionAssetInfo`.

The model records:

- exact runtime candidate metadata and approval state;
- host readiness requirements for WebGL, H.264 MP4 decode, local object URLs,
  blob media CSP, and GPU compositing;
- container facts from `metadata.vap`, including display/video dimensions,
  frame count, fps, duration, codec, audio/video presence, bounded-prefix state,
  and 1504 limit;
- fusion image/text runtime bindings derived from normalized resources/layers,
  including `srcId`, `srcTag`, fit/style/color metadata, placement samples,
  z-values, replacement requirements, and deterministic runtime binding keys;
- lifecycle/disposal/cancellation contract for the later runtime adapter.

Typed failure states:

- `unsupported`: missing WP3B gate;
- `missing_dependency`: dependency approval pending or rejected;
- `missing_resource`: replacement data missing or normalized resource/layer
  references are broken;
- `parse_precondition`: non-VAP asset, missing metadata, or missing video track;
- `ambiguous`: duplicate fusion runtime binding tags;
- `capability`: missing host capability or 1504-limit violation;
- `unsupported_feature`: non-H.264 codec or unknown fusion source type;
- `playback_failure`: reserved for the later runtime adapter.

Status behavior:

- `dependency_pending`: model is valid but runtime approval is not granted;
- `prepared`: dependency is approved, host capabilities are present, and no
  blocking issues remain;
- `blocked`: model exists but capability/unsupported/missing replacement issues
  block playback;
- `failed`: structural preconditions are invalid or ambiguous.

## Risks

- Official VAP Web upstream is stopped-maintenance even though the repository
  is not archived. Any adoption needs rollback and pinned-source evidence.
- Chromium/Electron H.264 decode can vary by build/platform; the first spike
  must prove actual packaged-runtime behavior before support claims.
- WebGL/GPU context loss, CSP, object URL cleanup, and target-container mutation
  are the main lifecycle risks. The preparation contract carries cancellation
  boundaries forward from the Lottie stale-renderer repairs.
- Fusion preview parity can diverge from native VAP implementations because
  fit/crop, text sizing, frame indexing, and mask placement are renderer- and
  platform-sensitive.
