## Runtime Audit

Approved source reference: Tencent VAP
`5d7c9f6789ccc5f384356001b71ef90c876cd95e`.

Package/source checks:

- `web/package.json` from the pinned source commit exactly matches installed
  `node_modules/video-animation-player/package.json`.
- `dist/index.d.ts` from the pinned source commit exactly matches the installed
  type entry.
- `src/video.ts`, `src/webgl-render-vap.ts`, `src/vap-frame-parser.ts`, and
  `dist/vap.js` differ from the pinned raw source in copyright header text
  (`Tencent` versus `THL A29 Limited, a Tencent company`) and line/build output
  details. The mismatch is recorded as an audit caveat rather than ignored.

Imported runtime behavior found in `dist` and `src`:

- Uses `window`, `document`, hidden `<video>`, canvas, `Image`, WebGL, and
  `requestVideoFrameCallback` / animation-frame fallback.
- `precacheSource` can use `XMLHttpRequest` and `URL.createObjectURL`.
- Config URLs ending in `.json` can use `XMLHttpRequest`.
- Fusion image values are assigned to `new Image().src`.
- `destroy()` removes video/canvas nodes and clears WebGL resources/listeners,
  but host-created object URLs must be revoked by the adapter.

Adapter mitigation:

- Dynamic import is hidden behind `VAP_WEB_PLAYBACK_WP3C_GATE`.
- The first data path passes `src` as a host-created `blob:` URL and passes the
  already-extracted embedded `vapc` object as `config`.
- `precache` is always `false`.
- Network/CDN fusion image replacements are rejected before local reads; image
  replacements must be `blob:` or `data:image/...`.
- Runtime construction is generation/cancellation-bound immediately before the
  renderer can touch a container.

## Implementation

`src/workbench/vap-inspection.ts`

- Preserves the extracted embedded `vapc` JSON object in
  `metadata.vap.config`, allowing playback to avoid config URL loading.

`src/workbench/vap-web-playback-adapter.ts`

- Adds the hidden VAP Web adapter and typed module loader.
- Consumes only prepared local object URL sources plus inspection metadata.
- Enforces gate, target, local `blob:` URL, successful inspection, dependency
  availability, preparation preconditions, and host capability readiness.
- Supports load, play, pause, seek, replay, loop, dispose.
- Destroys the runtime and releases object URLs on unload/reload/error/dispose.
- Ignores stale callbacks and releases object URLs if cancellation fires before
  runtime/container mutation.

`src/workbench/vap-preview-vertical.ts`

- Adds a hidden local open/drag/menu candidate session for VAP.
- Uses shared WP1 format detection and bounded host range reads.
- Runs VAP inspection and playback preparation before object URL creation.
- Creates host-owned object URLs only after safety/preparation checks pass.
- Binds generation identity across reads, detection, inspection, preparation,
  object URL creation, adapter load, recovery, and public controls.
- Keeps model paths redacted and `rendererHasFullPath: false`.

`src/tests/vap-web-playback-adapter.test.ts`

- Covers gate, local object URL/config load, controls, missing dependency,
  unsupported preparation states, destroy/revoke on runtime error, stale
  callback isolation, cancellation before runtime mutation, and default import
  resolution without invoking the real runtime.

`src/tests/vap-preview-vertical.test.ts`

- Covers hidden gate, bounded local open, fusion image/text binding, play,
  pause, seek, loop, dispose, missing fusion replacements, network replacement
  rejection, H.264-only readiness, stale object URL race, repeated load cleanup,
  runtime error cleanup/recovery, disposed controls, and formal `0.1.x`
  command-menu isolation.
