  display/video dimensions above the documented 1504 VAP compatibility limit.
- Updated VAP capability baseline to `partial` parse only, with explicit hidden
  inspection/readiness wording.

## VAP Playback Dependency Decision Evidence

Current refreshed facts on 2026-07-10:

- Official repository: `https://github.com/Tencent/vap`
- Official HEAD: `5d7c9f6789ccc5f384356001b71ef90c876cd95e`
- Maintenance: official README says the open-source repository has entered a
  stopped-maintenance state and will not continue updates.
- License: repository and Web source state MIT; repository `LICENSE.txt`
  also lists third-party/tooling license material including FFmpeg/GPL text,
  so tool/vendor redistribution needs separate legal review if copied.
- Official Web package documented by Tencent: `video-animation-player`.
- Web package source package.json: version `1.0.5`, main `dist/vap.js`,
  license `MIT`, runtime dependencies none listed, dev dependencies only.
- npm registry fact: latest `video-animation-player@1.0.5`, created
  `2020-09-07T08:15:14.640Z`, modified `2022-10-13T02:35:45.902Z`,
  unpacked size `3300932` bytes, integrity
  `sha512-KLz+uL6zojOYXEPFxL2AB0iKSLHnmKQPBnXmFWHpGtAdB6/j9EWCbWcUDCg0KVOBTFRHa2UfiSNK0y1I+LEfpA==`.
- Web README API: import default `Vap` from `video-animation-player`, create
  `new Vap(options)`, methods include `on`, `destroy`, `pause`, `play`,
  `setTime`; options include `container`, `src`, `config`, `width`, `height`,
  `fps`, `mute`, `loop`, `type`, `beginPoint`, `fontStyle`, `accurate`,
  `precache`, lifecycle callbacks, and arbitrary fusion ext params.
- Web source entry performs a WebGL capability check and throws when WebGL is
  unavailable.
- Official VAP model remains local MP4/H.264 plus embedded or sidecar JSON
  config; transparent output and fusion require video texture, image/text
  textures, and shader compositing.

Decision:

- Default recommendation: defer production adoption. The official package is
  old and upstream is stopped-maintenance; VAP playback also depends on
  Electron video codec behavior, WebGL/GPU lifecycle, local blob/object URL
  handling, CSP policy, and fusion synchronization.
- Conditional spike recommendation if Product Owner explicitly approves after
  CR/QA: exact-pin `video-animation-player@1.0.5`; use package entry
  `video-animation-player` / main `dist/vap.js`; load only through a hidden
  dynamic import in a 0.2 adapter; feed local MP4 as a bounded host-provided
  object URL and pass extracted `vapc` JSON as `config`; no CDN, no network
  fetch, no sidecar dependency as the normal path, no visible 0.1 UI.
- Rejected in this milestone: vendoring Tencent repo source, bundling official
  preview tools, FFmpeg/mp4edit/VapTool, codec/WASM/GPU helper packages,
  H.265 support claim, Windows support claim, or VAP export/generation.
- Bundle-size plan for the gated spike: install exact package only after
  approval, measure `node_modules/video-animation-player`, generated dynamic
  chunk raw/gzip/brotli size, Electron packaged size delta, and verify no
  network/CDN path or extra transitive runtime dependency is introduced.
- Rollback plan: revert the dependency/adapter spike commit; keep the parser
  inspection milestone because it has no runtime dependency or package change.
