## 5. Implementation

### Registry And Gate

- The default WP1 registry contains only `svga`, `lottie`, and `vap`.
- `MotionFormatProbeService.probe()` requires the exact
  `0.2-multiformat-preview-wp1` gate before reading bytes.
- No short-term host, menu, open, drag/drop, or playback module imports the new
  registry.

### Bounded Evidence

| Format | Hint evidence | Content evidence | Confirmed probe condition |
| --- | --- | --- | --- |
| SVGA | `.svga`, SVGA media types | valid zlib header | hint and full zlib CMF/FLG preconditions agree |
| Lottie | `.json`, Lottie/JSON media types | JSON object and minimal Lottie fields | `v`, `fr`, `ip`, `op`, `w`, `h`, and `layers` have expected shapes |
| VAP | `.mp4`/`.vap`, VAP/MP4 media types | MP4 `ftyp` and top-level `vapc` boxes | normal-size `ftyp` has minimum brand/version payload and `vapc` has non-empty payload |

- Probe byte inspection is capped at 256 KiB.
- Range-readable sources use `readRange(0, 262144)` even when caller metadata
  reports a smaller size. Declared-large sources without range reads fail
  closed with `parse_precondition`; there is still no production host call site
  in WP1.
- JSON probing checks only the minimal shape. MP4 probing checks only box
  structure and does not parse `vapc` payloads. Extended-size and size-zero MP4
  boxes remain candidates until a later parser strategy is approved.

### Typed Feedback

`MotionFormatProbeIssueCode` is restricted to:

- `unsupported`
- `missing_dependency`
- `parse_precondition`
- `ambiguous`

Issue messages and nested details use the shared local-path redaction helper.
Path-like source identities produce `[local path]`, while the safe basename is
available as `details.sourceName`.

The previous short-term redaction exports remain intact and delegate to the
same generic helper, so existing 0.1 callers keep the same API and behavior.

## 10. Risks And Limitations

- The 256 KiB prefix is intentionally a probe boundary. Large Lottie JSON and
  VAP files with `vapc` outside the prefix remain candidates until later parsers
  inspect them with an approved strategy.
- SVGA classification is bounded evidence, not protobuf parse success. Existing
  `SvgaFormatAdapter` remains the deep inspection boundary.
- Minimal Lottie shape recognition does not validate renderer compatibility or
  supported Lottie features.
- `vapc` presence does not validate its JSON payload, codec, alpha layout,
  fusion semantics, or playback.
- The WP1 probe now prefers `readRange` whenever available. For future host
  integration, untrusted sources should provide range reads or host-verified
  size metadata before relying on full `read()`.
- The generic path helper is shared with 0.1 redaction. Its existing API and
  behavior passed focused and full regression, but future changes remain a
  cross-lane review concern.

## 13. Project Retrospective

- Value assessment: High.
- Cost drivers: fail-closed evidence scoring needed separate candidate and
  detected states; redaction had to cover messages, issue paths, and nested
  details; shared infrastructure required full regression.
- Avoidable costs: the first source scan was broader than necessary; the WP0
  review plus exact authority section was enough to design the service.
- Product lessons: candidate detection is not format support and must stay
  invisible until later 0.2 UI and QA gates.
- Technical lessons: extension and media type are hints; content evidence and
  explicit preconditions prevent generic JSON/MP4 false positives.
- Design / interaction lessons: no visible format selector or unsupported-state
  UI belongs in WP1.
- Process lessons: keep PM authority updates in the governance lane and avoid
  duplicating them into implementation branches.
- Follow-up candidate for
  `docs/retrospectives/PROJECT_LESSONS_CANDIDATES.md`: Yes.
