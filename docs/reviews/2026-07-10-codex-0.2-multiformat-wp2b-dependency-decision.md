## 5. Candidate Details

### `lottie-web@5.13.0`

Facts:

- MIT license, no direct npm dependencies, no transitive dependency graph.
- Repository: `airbnb/lottie-web`, default branch `master`, not archived.
- GitHub API snapshot: 31,997 stars, 2,930 forks, 849 open issues, last push
  2025-09-01T09:01:31Z.
- Official README states `loadAnimation()` accepts `animationData` or `path`,
  and renderer can be `svg`, `canvas`, or `html`.
- Official README lists events including `data_failed`, `loaded_images`, and
  `DOMLoaded`, which are useful for typed playback failure feedback.
- Official README notes image/AI layers export to an images folder relative to
  the JSON, so Auto SVGA must keep its deterministic local asset resolver and
  must not pass arbitrary absolute paths or URLs.
- Official README lists support for precomps, shapes, solids, images, nulls,
  text, masks, time remapping, and expressions, but also states image
  sequences, videos, audio, and negative layer stretching are not supported.

Fit:

- Best default for WP2B because the current target is local Lottie JSON
  playback, not `.lottie` archives.
- SVG entry point is preferred for first spike because it is the common default
  renderer, easier to inspect with DOM/browser tooling, and avoids Canvas-only
  screenshot ambiguity during early evidence.
- Use `animationData`, not remote `path`, for the first spike to preserve
  offline behavior and avoid renderer-driven network fetches.
- Continue failing closed for expressions before renderer invocation. Lottie
  expressions must remain unsupported until a security decision says otherwise.

Risks:

- Large full package unpacked size due bundled player/plugin/examples.
- The minified SVG renderer file is about 243 KiB before gzip/brotli; actual
  production chunk size must be measured in this repo after approval.
- External images and fonts are renderer/environment dependent. Do not claim
  image/font fidelity until a deterministic local resolver and fixture matrix
  pass.
- Masks/mattes/effects/time remap may render differently across SVG/canvas.
  Early spike must report unsupported/untested features instead of treating
  successful playback of one fixture as product support.

### `@lottiefiles/dotlottie-web@0.77.0`

Facts:

- MIT license, no direct npm dependencies, no transitive dependency graph.
- Repository: `LottieFiles/dotlottie-web`, default branch `main`, not archived.
- GitHub API snapshot: 819 stars, 53 forks, 44 open issues, last push
  2026-07-09T08:13:44Z.
- Latest GitHub release tag:
  `@lottiefiles/dotlottie-web@0.77.0`, published 2026-07-09T07:51:19Z.
- Official docs call it the core web player, powered by `dotlottie-rs`, and
  show a canvas-based `DotLottie` API.
- Official docs list WebAssembly, Canvas 2D, and Fetch API as required, with
  Web Workers and OffscreenCanvas optional.
- README describes Software/Canvas2D, WebGL2, and experimental WebGPU
  backends, each with optimized WASM builds.
- jsDelivr metadata shows software WASM 1,799,926 bytes, WebGL WASM 1,924,408
  bytes, WebGPU WASM 1,961,029 bytes, plus JS entry files.

Fit:

- Strong non-default candidate for a later `.lottie` or WASM-renderer decision.
- Better current maintenance velocity than `lottie-web`, but that is not enough
  to override current scope and CSP risk.

Risks:

- Introduces WASM packaging, CSP, and Electron asset-loading work before we
  have owner approval for renderer adoption.
- `.lottie`, state machines, audio, slots, themes, and GPU backends are useful
  future scope but exceed the WP2B JSON-only gate.
- Worker/offscreen paths complicate deterministic screenshot and failure
  evidence.

## 7. Security, Privacy, CSP, And Offline Rules

Mandatory for any approved playback spike:

- Never pass production file paths or remote URLs directly to a renderer.
- Use `animationData` for inline JSON playback first.
- For external images, route through a deterministic local resolver that
  accepts only safe relative references already validated by WP2A.
- Reject or mark unsupported before playback when WP2A metadata finds
  expressions, embedded images, missing images, unsafe paths, duplicate ids, or
  unresolved layer references.
- Do not enable `unsafe-eval`, remote `connect-src`, remote `img-src`, or
  `wasm-unsafe-eval` as part of `lottie-web` adoption.
- Do not upload Lottie JSON, images, fonts, logs, or screenshots to external
  services.
- Store renderer failures as typed local feedback with redacted paths.

## 8. Synthetic Fixture And Feature Matrix

All fixtures must be synthetic and committed only when they are small JSON or
programmatically generated test assets. No production Lottie/image/video/frame
asset may be committed.

| Fixture | Purpose | Expected WP2B spike result |
| --- | --- | --- |
| Inline shape, no assets | Baseline SVG playback | Plays, frame timing observable, no network |
| Inline text with declared font metadata | Text candidate boundary | Plays or degrades with font warning; no support claim |
| Relative local image reference with tiny generated PNG | Resolver path | Plays only through deterministic resolver |
| Missing image reference | Typed failure | `parse_precondition` or playback precondition, no renderer call |
| Parent traversal / absolute / file URL image | Security boundary | Fail closed before renderer call |
| Expression marker | Security boundary | Fail closed before renderer call |
| Mask/matte simple synthetic | Feature matrix | Attempt only if Product Owner approves; unsupported/partial if mismatch |
| Effect/time remap/3D marker | Unsupported markers | Do not claim support; typed warning/failure |
| Large dimensions / high layer count | Performance guard | Fail or warn based on bounded thresholds |
| Malformed JSON / non-Lottie JSON | Regression | Reuse WP1/WP2A typed failure |

## 9. Bounded WP2B Playback Spike

Proposed scope if Product Owner approves:

- Add exact dependency `lottie-web@5.13.0`.
- Add hidden 0.2-only adapter module, no visible UI.
- Use SVG renderer entry `lottie-web/build/player/lottie_svg`.
- Render only synthetic inline JSON and safe local resolver fixtures in a
  headless/browser test harness.
- Collect deterministic evidence:
  - renderer loads without network;
  - DOM/SVG root is non-empty;
  - frame advances or events fire;
  - typed load/render failures map to existing feedback;
  - no 0.1 open/menu/drag/playback path imports the renderer;
  - bundle measurement proves 0.1 isolation.

Success criteria:

- `npm run build` and focused renderer tests pass.
- Synthetic inline shape fixture plays in hidden harness.
- Unsafe/missing/expression fixtures fail before renderer call.
- Bundle measurement recorded with raw/gzip/brotli and 0.1 chunk isolation.
- No visible 0.2 entry, no package promotion, no production support claim.

Failure criteria:

- Dependency leaks into 0.1 chunks or visible 0.1 commands.
- Renderer requires `unsafe-eval`, `wasm-unsafe-eval`, remote fetch, or broad
  CSP relaxation.
- External image resolver cannot stay deterministic and path-redacted.
- Playback success cannot be proven with nonblank/moving/event evidence.

Non-claims:

- Not Lottie production support.
- Not `.lottie` support.
- Not VAP support.
- Not text/image replacement support.
- Not export/save/edit support.
- Not owner-visible package readiness.

## 10. Decision

Recommendation: approve `lottie-web@5.13.0` for WP2B playback spike only.

Exact adoption request:

- Dependency: `lottie-web@5.13.0`.
- Version policy: exact pin.
- Initial entry point: `lottie-web/build/player/lottie_svg`.
- Import style: 0.2-gated dynamic import in the hidden playback adapter.
- Renderer mode: SVG.
- Data path: `animationData` only for the first fixture.
- External image path: deferred until deterministic local resolver is approved
  inside the spike.

Reject/defer:

- Defer `@lottiefiles/dotlottie-web@0.77.0` for this WP because `.lottie` and
  WASM/GPU/worker scope are not needed for the first JSON playback spike.
- Reject unscoped `dotlottie-web`; npm registry has no such package.

Approval required:

- Product Owner approval is required before any install, `package.json`
  change, lockfile change, renderer import, playback code, UI, package, or
  promotion.
