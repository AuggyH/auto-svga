## Implementation

- Added a hidden `0.2` Lottie preview vertical session with explicit gate
  `0.2-hidden-lottie-preview-vertical`.
- Added a host contract that uses local file stat plus bounded range reads for
  detection, inspection, and playback JSON.
- Routed local file-button, menu-open, and drag-drop sources through the WP1
  registry before Lottie-specific inspection.
- Reused WP2A Lottie JSON inspection for normalized version, dimensions,
  timing, layers, image assets, text candidates, and unsupported markers.
- Added a hidden normalized surface with overview, layers, assets, replaceable
  image/text candidates, unsupported markers, playback state, and typed
  failures.
- Added deterministic adjacent image resolution for relative image references.
  Resolved images are inlined as `data:` URIs inside renderer `animationData`
  so the renderer receives no local filesystem path.
- Kept missing, unsafe, oversized, unsupported media, font, embedded, and
  unsupported-feature cases fail-closed before renderer load.
- Extended the approved Lottie SVG adapter with an opt-in
  `allowResolvedImageResources` switch. Default WP2B behavior remains
  animationData-only and still rejects image/font resources.
- Added a hidden Node host adapter for bounded local JSON and adjacent-resource
  reads. It is not imported by the formal `0.1.x` host.

## Risks

- This is hidden model/host evidence, not owner-visible DOM playback evidence.
- Data URI inlining is bounded by the 256 KiB animationData read limit; larger
  image sets may need a later renderer-resource strategy.
- Fonts remain metadata-only and block playback.
- Unsupported Lottie features block playback rather than silently rendering.
- Future visible UI must still pass Product/Code Review/QA gates before any
  support claim.

## Project Retrospective

- Value assessment: High.
- Cost drivers: vertical scope crossed registry, host reads, JSON inspection,
  resource safety, playback lifecycle, and hidden surface modeling.
- Avoidable costs: none obvious; this milestone intentionally replaced several
  micro-review slices with one vertical.
- Product lesson: hidden vertical evidence can advance `0.2.x` without changing
  owner-visible `0.1.x`.
- Technical lesson: adjacent Lottie image support should be proven by a host
  resolver that inlines safe resources instead of letting the renderer chase
  local paths.
- Design lesson: candidate overview/assets/replaceable/error states can be
  normalized before any visible format selector exists.
- Process lesson: keep frozen QA heads immutable and move successor work to a
  descendant branch.
- Token usage source: unavailable.
