## Feasibility And Decision

- Research or feasibility summary: The audited official runtime supports local playback, inspection, and session-level text/image replacement. It does not provide a general serialization API for those runtime changes. Explicit PAG/WASM/WebGL destruction and memory-settle validation are required.
- Dependencies: pinned runtime/license/redistribution review; safe editable text/image fixtures; runtime closure and offline packaging; material/evidence plan.
- Risks: runtime/version feature drift, WASM/WebGL memory growth, replacement APIs reporting success without visible pixels, fonts/resources, and unsupported PAG feature ambiguity.
- Alternatives considered: Add PAG to the current 0.2.0 candidate, rejected because it would reopen a closing local-stable baseline; start with exporter work, rejected because preview/runtime and export are different risk boundaries.
- Final decision: Route a bounded preview/runtime ticket after the current local-stable gate. Keep AEB-to-PAG export separate.
