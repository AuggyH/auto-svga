## Terminology Boundary

- `图层` means a designer/editor layer in source or Edit mode. It is what a
  designer thinks of as a layer in AE-like authoring or in Auto SVGA's layer
  panel.
- SVGA `SpriteEntity` is a runtime playback object after export. It is bound to
  an `imageKey` and a list of frame states. One designer layer may export to one
  sprite, multiple sprites, or a sequence of many sprites depending on the
  converter and sequence-frame structure.
- SVGA `FrameEntity` is a per-frame state record for a sprite, including values
  such as alpha and transform. A high FrameEntity count means the file contains
  many runtime state records, not that the designer created thousands of
  visible layers.
- User-facing copy should explain this as `运行时结构复杂`, `动画对象过多`, or
  `动画帧记录过多`; it must not say `图层过多` for sprite or FrameEntity counts.

## Metric Calculation Guide

- `运行对象数`: count parsed SVGA runtime sprite records, equivalent to
  `MovieEntity.sprites.length` in the decoded SVGA structure.
- `动画帧记录数`: sum all runtime sprite frame-state records, equivalent to
  `sum(sprite.frames.length)`.
- `活跃绘制峰值/平均`: for each frame index, count runtime sprites whose frame
  state is visible under the active threshold, then report maximum and average.
  Without a selected target-player profile, use `alpha > 0` as the generic
  alpha-positive signal. Player-specific thresholds must be reported separately.
- `不可见记录占比`: invisible frame-state records divided by total frame-state
  records. Generic invisible means alpha is zero or absent in a way that renders
  invisible. Low-alpha pruning thresholds require a target-player profile.
- `序列帧展开风险`: detect repeated numbered sequence-frame imageKeys or
  sequence-like runtime object groups. Flag risk when unique image count is low
  but runtime object and frame-record counts are high.
- `估算运行时结构内存`: advisory estimate derived from frame-record count plus
  runtime-object overhead, calibrated by known production cases when available.
  It must be labelled as estimated, not measured phone memory.

## Right Information Display Policy

- Default summary should include the common, decision-useful fields: risk
  level, `运行对象数`, `动画帧记录数`, estimated runtime-structure memory/risk
  text, and the specific field that explains the current risk or optimization
  candidate.
- `更多信息` may include secondary diagnostics such as alpha-positive record
  count, invisible/low-alpha ratios, visible peak/average, sequence fanout group
  count, top repeated sequence groups, longest runtime-object frame list, active
  target-player threshold profile, and calculation notes.
- Risk-causing fields are never secondary-only. If a collapsed detail causes
  medium/high risk, warning copy, or an optimization candidate, promote that
  field into the visible summary with a short explanation.
- Low-risk files may keep most detail collapsed as long as the summary clearly
  communicates that no runtime-structure issue is currently detected.
