## Product Scope

- PRD anchor: `docs/product/PRODUCT_ROADMAP.md`, `Planned 0.5.x: VAP Generation And Format Conversion`
- Related product brief or review: `docs/research/vap-official-repository-research.md`
- Target product version: Auto SVGA `0.5.x`
- Product phase: Other
- Requirement type: Production workflow
- Current behavior or limitation: Auto SVGA can preview and runtime-replace VAP fusion targets but cannot generate a VAP file.
- Intended behavior: Normalize a video or ordered frame sequence into one local VAP project, edit mask/fusion/resolution parameters with live preview, generate through a pinned official VapTool adapter, and validate the output.
- User value: Produce compatible VAP assets through one inspectable and repeatable desktop workflow.
- Explicit non-goals: general video editing, unrestricted compositing/timeline, cloud transcoding, destructive source-folder edits, silent fusion acceptance, or accepting parseable bytes without real playback.

## Feasibility And Decision

- Research or feasibility summary: Official VapTool accepts ordered frames, FPS, quality/codec, audio, and fusion-mask information and emits MP4 plus `vapc`. Official guidance identifies 1504 and 16-pixel alignment constraints. Upstream maintenance has stopped, so the adapter, dependency closure, fixtures, and compatibility profile must be pinned and owned locally.
- Dependencies: VapTool/license/redistribution ADR; local video decode strategy; frame/mask model; fusion schema; output profile; real runtime and device compatibility fixtures.
- Risks: codec/tool installation, platform drift, alpha/mask layout, fusion jitter, frame ordering, quality loss on downscale, H.264/H.265 compatibility, long jobs, cancellation, and staging residue.
- Alternatives considered: reimplement the whole encoder initially, rejected in favor of a controlled official-tool adapter; treat videos and sequences as separate products, rejected because both should converge on one normalized project.
- Final decision: Build one transactional pipeline with two intake paths and a pinned official-tool backend.

| AC5 | Output resolution is editable and export guarantees `max(videoW, videoH) <= 1504`, preserves aspect, avoids default upscaling, and reports 16-pixel alignment and quality impact. | Boundary matrix around 1504, portrait/landscape/square, and output config readback. |
| AC6 | A pinned official VapTool adapter receives exact normalized inputs and generates a VAP MP4 with valid embedded `vapc`. | Tool/runtime identity, command receipt, hash-bound stage records, and MP4 box validation. |
| AC7 | Generated output reopens and plays in the pinned runtime with correct alpha, masks, fusion replacement, timing, dimensions, and pause stability. | Real playback, pixels, time advance, fusion replacement, and output facts. |
| AC8 | Cancellation/failure keeps source and prior valid output unchanged; staging is recoverable and cleanup leaves no process/listener/residue. | Transaction, retry, rollback, and cleanup proof. |
