# Weekly Project Retrospectives

Weekly retrospectives aggregate recent task reviews, task-ledger entries,
commits, blockers, validation gaps, design decisions, coordination issues, and
token cost.

Use the format `YYYY-WW.md`, for example `2026-W28.md`.

Each weekly review should include:

- reviewed date range
- source inputs
- completed work by lane
- unresolved blockers and risks
- product, technical, UI/UX, validation, release, coordination, and token-cost
  lessons
- high-value high-cost work and why it was justified
- low-value high-cost work and how to avoid it
- lessons to add or promote in
  `docs/retrospectives/PROJECT_LESSONS_CANDIDATES.md`

Do not paste raw chat history, long logs, full diffs, or production assets.
