# Monthly Project Experience Distillation

Monthly retrospectives distill repeated weekly lessons into durable project
guidance.

Use the format `YYYY-MM.md`, for example `2026-07.md`. A monthly pass may also
run after four weekly reviews even if the calendar month has not ended.

Each monthly review should include:

- source weekly retrospectives and task-ledger range
- repeated lessons
- rejected or stale lessons
- updates made to `docs/retrospectives/PROJECT_EXPERIENCE_GUIDE.md`
- candidate lessons left for more evidence
- changes recommended for task templates, automation prompts, validation
  strategy, or coordination rules

Promote a lesson only when it repeated, prevented real risk, materially reduced
cost or rework, or was explicitly confirmed by the Product Owner.
