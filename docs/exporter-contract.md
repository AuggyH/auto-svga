# Exporter Contract

This repository exports an intermediate protocol and a minimal real SVGA protobuf + zlib binary.

The real SVGA exporter must read:

- `project.json`
- `svga-map.json`
- `output/assets/*`

## Coordinate Convention

The coordinate convention is frozen:

- `layer.transform.x` / `layer.transform.y` is the layer anchor position in canvas coordinates.
- `layer.anchor.x` / `layer.anchor.y` is the anchor position in the layer local coordinate system.
- `rotation` and `scaleX` / `scaleY` are applied around `anchor`.
- `preview-renderer` and future exporters must use the same coordinate convention.

## Project Input

The exporter should consume image layers and frame-based animations:

- `assets`: image resources copied or generated by the build step
- `layers`: concrete image layers only
- `animations`: frame-based keyframes targeting layer IDs
- `mask`: optional alpha-mask protocol on layers
- `fallbackBlendMode` and `fallbackOpacityMultiplier`: compatibility strategy for unsupported blend modes

The exporter must not understand or branch on template semantics such as:

- `breathing_glow`
- `metal_edge_sweep`
- `gem_twinkle`

Templates are build-time authoring concepts only. By export time they must already be expanded into image layers, sprites, transforms, masks, and keyframes.

## Minimal SVGA Export

The current exporter supports:

- image sprites
- `exportAssetPath`
- alpha
- layout
- transform matrix
- frame-by-frame keyframes
- baked masked assets

The current exporter does not support:

- runtime mask
- text
- dynamic avatar replacement
- complex shapes
- audio
- non-linear easing

Easing values are currently linearized during protobuf export.

## SVGA Map

`svga-map.json` provides a sprite-oriented mapping preview:

- `spriteId`
- `assetPath`
- `zIndex`
- `transform`
- `keyframes`
- `blendMode`
- `fallbackBlendMode`
- `fallbackOpacityMultiplier`
- `mask`
- `bakedMaskAssetPath`
- `replaceable`

`bakedMaskAssetPath` is provided for exporters or previews that cannot evaluate alpha masks directly.

## Replaceable Sprites

In the current avatar frame MVP, all sprites are non-replaceable.

Do not mark `frame_base` as replaceable. If a later phase supports dynamic user avatar replacement, add a dedicated `avatar_slot` or `user_avatar` sprite and mark that sprite as replaceable.

## Exporter Readiness

`report.json.exporterReady` should be true before attempting real binary export. It requires:

- no unsupported layer types
- no template-only animations
- all animation target layer IDs exist
- all layer asset IDs exist
- all keyframes are in range
- all non-normal blend modes have fallback fields
- replaceable sprite semantics are valid
- masked layers have protocol fields and baked mask assets
