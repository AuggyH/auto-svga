# Auto SVGA Agent Loop Repo Audit

Date: 2026-06-19

## Scope

This audit prepares the repository for an Agent Loop. It only reads the repo, runs existing safe local commands, and creates documentation under `docs/loop/`.

No runtime code, tests, dependencies, or configuration were changed.

## Technical Stack

- Runtime: Node.js with ESM modules.
- Language: TypeScript, compiled with `tsc -p tsconfig.json`.
- Test runner: Node built-in `node --test`.
- CLI entrypoint: `src/cli.ts`, compiled to `dist/cli.js`.
- Package binary: `svga-avatar-frame -> ./dist/cli.js`.
- Core dependencies: `protobufjs`, `fast-png`.
- Web preview: local Node HTTP server under `tools/svga-player-preview/`.
- Local launcher: Node script under `tools/launch-local-preview.mjs`.
- Desktop experiments: isolated Electron prototype under `tools/electron-prototype/`.
- SVGA binary export: protobuf schema at `proto/svga.proto` plus zlib/protobuf validation.

No root `lint` or formatter script is currently defined.

## Key Execution Entrypoints

Root package scripts:

- `npm run build`: TypeScript build.
- `npm test`: full root test suite via `npm run test:all`.
- `npm run test:mvp`: focused MVP planner test.
- `npm run validate:example`: validates `examples/avatar_frame_basic`.
- `npm run build:example`: generates example outputs.
- `npm run export:example`: generates example SVGA output.
- `npm run preview:player`: starts the Web playback preview server.
- `npm run local:preview`: starts the one-click local browser workflow launcher.
- `npm run autosvga:plan|preview|report|export|package|accept`: CLI command shorthands requiring a job path.

Isolated desktop prototype scripts:

- `npm --prefix tools/electron-prototype run spike:electron:test`
- `npm --prefix tools/electron-prototype/experiments/svga-web run spike:svga-web:test`
- `npm --prefix tools/electron-prototype/experiments/svga-web run spike:svga-web:smoke`

These are not default runtime paths. They write ignored `.runtime` folders.

## Directory Structure

- `src/commands/`: CLI command implementations.
- `src/mvp/`: avatar-frame MVP pipeline.
- `src/core/`: shared core helpers.
- `src/exporters/`: exporter adapter boundaries and SVGA exporter.
- `src/preview/`: preview rendering support.
- `src/hosts/`: host-side source inspection, hashing, and analyzer composition.
- `src/workbench/`: FormatAdapter, MotionAssetInfo, spec checking, Motion Asset Audit, format recommendation primitives.
- `src/tests/`: TypeScript tests compiled into `dist/tests/`.
- `proto/`: SVGA protobuf schema.
- `schemas/`: JSON schemas.
- `templates/`: avatar-frame template definitions.
- `examples/avatar_frame_basic/`: example input workspace.
- `tools/svga-player-preview/`: browser playback and inspection preview.
- `tools/electron-prototype/`: isolated desktop prototype experiments.
- `docs/`: technical specs, decisions, reviews, product notes, templates.
- `codex-skills/`: repository-local task guidance skills.
- `dist/`: compiled output, generated by build.
- `jobs/`, `outputs/`, `.artifacts/`: local runtime or generated workspaces; not suitable for source-of-truth loop assertions.

## Key Project Contracts

- `project/project.json` remains the source of truth for preview/export pipeline work.
- `svgaExport.success` is separate from exporter readiness.
- Protobuf inflate/decode validates binary structure, not visual playback quality.
- Real SVGA visual success still needs player-based or manual confirmation.
- Motion Asset Audit report contract is versioned and currently v1.
- Role-aware transparent-padding diagnostics are advisory; production gate migration is frozen until labeled evidence exists.
- Electron work is isolated prototype work only; browser workflow remains the stable rollback.

## External And Host Boundaries

- Root tests are local and do not require production credentials.
- The normal browser preview historically uses CDN player assets for current Web playback flow; Electron prototype experiments use local vendored player assets.
- Isolated Electron prototype work depends on local `tools/electron-prototype/node_modules`, not root production runtime.
- Some preview generation paths can depend on host tools such as `ffmpeg`; tests include fallback coverage when unavailable.
- Real external SVGA sample parity is intentionally not part of source control and should stay out of Agent Loop default validation.
