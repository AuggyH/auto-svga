## Summary

P4 implements bounded multi-resource embedded PNG replacement for the isolated
Electron desktop prototype. It adds two-resource replacement, undo/redo, reset
selected, reset all, save-point dirty state, Save As, reopened export proof,
schemaVersion 3 per-resource round-trip validation, and visible review upload
packaging.

P4 remains `HUMAN_REQUIRED` because final product acceptance must be made by the
product owner after reviewing the sealed packet and upload ZIP.

## Requirement Checks

- P3 closure: complete before P4 branch start.
- Canonical multi-resource fixture: synthetic fixture contains three PNG
  resources, two visible sprite-backed resources, and one untouched resource.
- Multi-resource replacement: two different resource keys are replaced in one
  session and both are independently verified.
- Undo / redo: replace, reset selected, reset all, redo, and redo branch
  truncation are covered by targeted history tests and Electron smoke.
- Dirty / save point: dirty is based on replacement digest versus saved digest,
  not replacement count.
- Round-trip integrity: schemaVersion 3 report validates each replacement,
  untouched resources, sprite references, and invariant checks.
- Original immutability: source SHA-256 is compared before and after editing.
- Security boundary: renderer still has no filesystem access; Electron security
  tests keep contextIsolation, sandbox, nodeIntegration=false, local-only assets,
  and no CDN.
- Scope: no text editing, transform editing, crop, resize, conversion,
  installer, release, AI, cloud, account, or new format parser.
