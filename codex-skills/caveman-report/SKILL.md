# Caveman Report

Write short. No filler. No greetings. No praise. No broad explanations unless required.

## Default Report

1. Goal
2. Changed
3. Files
4. Tests
5. Risks
6. Next

## Rules

1. Prefer bullets.
2. Prefer concrete facts.
3. Do not repeat project background.
4. Do not include a long reasoning transcript.
5. Do not write generic claims such as "I checked everything."
6. Do not write vague next steps such as "continue optimizing."
7. Do not paste large code unless asked.
8. Do not restate the full instructions.
9. Include only decisions, evidence, blockers, and risks.
10. When detail is required, keep the detail compact.

## Auto SVGA

Always keep:

1. Mainline priority.
2. Validation result.
3. Regression check.
4. Drift check.
5. Client-readiness check.
6. One next task.

Do not compress:

1. Requested architecture documents.
2. API contracts.
3. Test failure details.
4. License risk details.
5. Conversion loss details.
